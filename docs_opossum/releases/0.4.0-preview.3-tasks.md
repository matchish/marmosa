# 0.4.0-preview.3 Release Tasks

**Type:** Patch — bug fixes + sample cleanup. No breaking changes.  
**Builds on:** 0.4.0-preview.2  
**Reason for separate release:** The two library correctness fixes (especially the
`UpdateAsync` duplicate-event bug) are important enough to ship independently, without
forcing consumers to simultaneously deal with the breaking `AutoRebuild` enum change
that arrives in 0.4.1-preview.1.

## What Is In This Release

| Change | Type | Scope |
|--------|------|-------|
| `UpdateAsync` correctness bug — duplicate event application + checkpoint regression | Bug fix | Library core |
| `RebuildAsync` / `RebuildAllAsync` — checkpoint not written on empty store | Bug fix | Library core |
| `CourseBookOrderHistory` projection removed | Cleanup | Sample app only |

### Bug Fix Details

**`UpdateAsync` correctness bug**

`ProjectionManager.UpdateAsync` now reads each projection's checkpoint *inside* the
per-projection lock and filters incoming events by **both event type and position**
(`e.Position > checkpoint`). Previously events were applied to every projection without
a position guard, causing silent re-application of already-processed events and a
checkpoint regression cascade. The checkpoint is now always advanced to `batchMax`.

**`RebuildAsync` empty-store bug**

`RebuildAsync` and `RebuildAllAsync` no longer leave a projection in the "never rebuilt"
state when the event store is empty. The checkpoint file is now always written after a
rebuild (position 0 for an empty store). `RebuildAllAsync(forceRebuild: false)` uses
`File.Exists` on the checkpoint file to distinguish "rebuilt on empty store" from "truly
never rebuilt", preventing infinite re-queue on every daemon restart.

**`CourseBookOrderHistory` removal**

The projection used `evt.Position.ToString()` as its key selector (one file per purchase
event = O(Events) cardinality). On a Large-seeded database this produced 700,000+ files
and never completed rebuilding. The concept is already covered by `StudentPurchasedBooks`
(O(Students)). See `docs/lessons-learned/course-book-order-history-projection-mistake.md`.
