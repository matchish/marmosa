## Context

The original plan (`0.4.1-preview.1-tasks.md`) was written before the Scalable Projection
Rebuild was implemented. That refactor has since been fully completed (Phases 1–6) and
introduced several changes that affect this plan:

1. **Rebuild methods moved** from `IProjectionManager` to `IProjectionRebuilder`.
   `ProjectionDaemon` now injects `IProjectionRebuilder` and calls it for all rebuild
   operations.
2. **Crash recovery added.** `ResumeInterruptedRebuildsAsync` is called on startup inside
   the `if (_options.EnableAutoRebuild)` block. This call must be moved **outside** the
   mode switch — crash recovery must run unconditionally (a manually-triggered rebuild
   that was interrupted by a crash must be recovered even when auto-rebuild is `None`).
3. **New source files** reference `EnableAutoRebuild` that did not exist in the original
   plan (crash recovery tests, end-to-end tests, fixtures).
4. **New documentation** references `EnableAutoRebuild` that was not in the original plan
   (configuration-validation.md, testing-configuration-override-fix.md,
   implementation-summary-parallel-projection-rebuilding.md).
5. **Version target changed.** No more 0.4.x releases. This work ships as part of
   0.5.0-preview.1 alongside the scalable rebuild.

## API Design (unchanged from original plan)

### New enum: `AutoRebuildMode`

```csharp
/// <summary>
/// Controls how the projection daemon handles missing or stale projections at startup.
/// </summary>
public enum AutoRebuildMode
{
    /// <summary>
    /// The daemon starts but never triggers a rebuild on startup.
    /// Use in production when projections are managed manually or via admin endpoints.
    /// Was: EnableAutoRebuild = false
    /// </summary>
    None,

    /// <summary>
    /// Only projections whose checkpoint file is absent are rebuilt.
    /// Projections that were already rebuilt (even against an empty store) are skipped.
    /// This is the recommended default for most deployments.
    /// Was: EnableAutoRebuild = true (the only mode previously available)
    /// </summary>
    MissingCheckpointsOnly,

    /// <summary>
    /// All projections are rebuilt from scratch on every startup, regardless of checkpoints.
    /// Use in development when iterating on projection logic and wanting a clean slate
    /// on each restart, or after deploying a projection fix that requires a full replay.
    /// Was: not expressible with a boolean.
    /// </summary>
    ForceFullRebuild
}
```

### Updated property on `ProjectionOptions`

```csharp
// Before
public bool EnableAutoRebuild { get; set; } = true;

// After
public AutoRebuildMode AutoRebuild { get; set; } = AutoRebuildMode.MissingCheckpointsOnly;
```

### Migration guide for existing users

| Old config / code | New equivalent |
|---|---|
| `"EnableAutoRebuild": true` | `"AutoRebuild": "MissingCheckpointsOnly"` |
| `"EnableAutoRebuild": false` | `"AutoRebuild": "None"` |
| `options.EnableAutoRebuild = true;` | `options.AutoRebuild = AutoRebuildMode.MissingCheckpointsOnly;` |
| `options.EnableAutoRebuild = false;` | `options.AutoRebuild = AutoRebuildMode.None;` |

Default behaviour is **unchanged** — `MissingCheckpointsOnly` is the new default,
which is exactly what `true` did before.

## Task 3 — Critical Adjustment: `ProjectionDaemon.cs`

This is the most significantly changed task compared to the original plan. Three issues
required rework:

### Issue A: `ResumeInterruptedRebuildsAsync` must be unconditional

The scalable rebuild (P3-T6) placed `ResumeInterruptedRebuildsAsync` **inside** the
`if (_options.EnableAutoRebuild)` block. With the enum, crash recovery must run regardless
of the `AutoRebuild` mode. A manually-triggered rebuild that was interrupted should be
resumed even when `AutoRebuild = None`.

The architecture document §8.2 (Crash Recovery on Restart) shows crash recovery as
**Step 1 before the auto-rebuild check**. This is the correct design.

### Issue B: `_projectionManager.RebuildAllAsync` no longer exists

The original plan's `ForceFullRebuild` case called `_projectionManager.RebuildAllAsync(...)`.
After Phase 1 of the scalable rebuild, rebuild methods live on `IProjectionRebuilder`.

### Issue C: New `RebuildAllProjectionsAsync` private method needed

The `ForceFullRebuild` case needs a new private method analogous to the existing
`RebuildMissingProjectionsAsync` but calling `_projectionRebuilder.RebuildAllAsync(forceRebuild: true, ...)`.

### Corrected `ProjectionDaemon.ExecuteAsync` startup block

```csharp
// Always resume interrupted rebuilds (crash recovery) — regardless of AutoRebuild mode.
// A manually-triggered rebuild that was interrupted by a crash must be recovered even
// when auto-rebuild is off.
await _projectionRebuilder.ResumeInterruptedRebuildsAsync(stoppingToken).ConfigureAwait(false);

switch (_options.AutoRebuild)
{
    case AutoRebuildMode.MissingCheckpointsOnly:
        await RebuildMissingProjectionsAsync(stoppingToken).ConfigureAwait(false);
        break;
    case AutoRebuildMode.ForceFullRebuild:
        await RebuildAllProjectionsAsync(stoppingToken).ConfigureAwait(false);
        break;
    case AutoRebuildMode.None:
    default:
        break; // daemon starts but does not trigger any rebuild
}
```

### New private method: `RebuildAllProjectionsAsync`

```csharp
private async Task RebuildAllProjectionsAsync(CancellationToken cancellationToken)
{
    LogCheckingForRebuilds();

    var result = await _projectionRebuilder.RebuildAllAsync(
        forceRebuild: true,
        cancellationToken).ConfigureAwait(false);

    if (result.TotalRebuilt == 0)
    {
        LogAllProjectionsUpToDate();
        return;
    }

    if (result.Success)
    {
        LogRebuildSucceeded(result.TotalRebuilt, result.Duration);
    }
    else
    {
        LogRebuildWithFailures(
            result.TotalRebuilt,
            result.Details.Count - result.TotalRebuilt,
            string.Join(", ", result.FailedProjections));
    }
}
```
