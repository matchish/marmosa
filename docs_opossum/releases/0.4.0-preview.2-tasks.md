## Task Detail

### Task 1 ‚Äî CHANGELOG.md

The entire file currently only contains the 20260303 benchmark entry under `[Unreleased]`.
All feature and fix work must be documented, and the section renamed to the release version.

**`[Unreleased]` section should become `[0.4.0-preview.2] - 2026-03-03`.**

#### Added
- `IEventStore.ReadLastAsync(Query, CancellationToken)` ‚Äî returns the single highest-position
  matching event in O(1) file reads (one index lookup + one file read); scales from 100 to
  10,000 events in 799‚Äì1,105 Œºs; used for consecutive-sequence patterns (invoice numbering).
- N-ary `BuildDecisionModelAsync` overload ‚Äî accepts
  `IReadOnlyList<IDecisionProjection<TState>>` for runtime-variable homogeneous projections
  (e.g. shopping-cart price validation spanning an arbitrary number of items in a single read).
- `TimeProvider` constructor overload on `DecisionProjection<TState>` ‚Äî enables fully
  deterministic unit testing of time-dependent projections without sleeping; use
  `FakeTimeProvider` from `Microsoft.Extensions.TimeProvider.Testing`.
- **Sample: Opt-In Token pattern** (`ExamRegistration/`) ‚Äî server-generated single-use exam
  registration tokens (issue / redeem / revoke); event store as token registry; single
  `ExamTokenStatus` enum projection replaces two-bool pattern and handles revocation as a
  third state.
- **Sample: Dynamic Product Price ‚Äî all three features** (`CourseBookPurchase/`):
  Feature 1 ‚Äî current price without grace period;
  Feature 2 ‚Äî configurable grace period with `TimeProvider` for testability;
  Feature 3 ‚Äî shopping-cart N-ary overload (single read + single `AppendCondition` spanning
  all items).
- **Sample: Prevent Record Duplication** (`CourseAnnouncement/`) ‚Äî client-generated
  idempotency token; two-projection `BuildDecisionModelAsync`; token freed on retraction
  (event fold handles reuse automatically).
- **Sample: Invoice Number / Consecutive Sequence** (`InvoiceCreation/`) ‚Äî gap-free invoice
  numbering with `ReadLastAsync`; bootstrap race closed via `AfterSequencePosition = null`
  on first invoice.
- **Sample: Event-Sourced Aggregate** (`CourseAggregate/`) ‚Äî DCB tag-scoped `AppendCondition`
  in repository replaces named-stream lock; same events shared with Decision Model endpoints;
  side-by-side comparison documented in README.
- **Sample: `CourseBuyersProjection`** ‚Äî read model that tracks every student who purchased
  each course book.
- **Sample: Paging** ‚Äî `PaginatedResponse<T>` returned from list endpoints; `SortOrder` enum
  for ascending/descending control.
- **Data Seeder** (`Opossum.Samples.DataSeeder`) ‚Äî standalone CLI tool that seeds the
  `CourseManagement` sample with realistic data across all scenarios; includes progress bar.

#### Fixed
- Zero-allocation hot path restored in incremental projection updates ‚Äî dead
  `GetCheckpointAsync` call removed and in-memory checkpoint cache added; 55‚Äì65% speedup;
  **0 B allocated per incremental update** (was 11‚Äì16 KB).
- Append regression resolved for large no-flush batches ‚Äî batch-50 ‚àí13%, batch-100 ‚àí12%
  vs 0.4.0-preview.1.
- Projection rebuild admin endpoint ‚Äî now correctly targets only the named projections
  supplied; previously rebuilt all projections regardless of input.
- Test flakiness fixed in concurrent-append integration tests.

#### Performance (benchmark run 2026-03-03, rerun1 verified)
- Incremental projection update: **3.7 Œºs / 0 B** (1 event) and **4.5 Œºs / 0 B** (10 events)
  ‚Äî zero-allocation hot path confirmed in rerun1.
- `ReadLastAsync`: 799 Œºs (100 events) ‚Üí 1,105 Œºs (10,000 events) ‚Äî near-O(1) confirmed;
  192√ó faster than full `ReadAsync` at 10K events.
- Single-event flush append: ‚àí5% vs preview.1 (16.4 ms ‚Üí **~61 events/sec**).
- Batch-10 flush append: ‚àí3% vs preview.1 (124 ms ‚Üí **~81 events/sec**).
- High-selectivity tag query: 553 Œºs ‚Üí **~501 Œºs** (‚àí9%).
